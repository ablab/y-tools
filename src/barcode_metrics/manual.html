1. What is BarcodeQUAST?

BarcodeQUAST is application for quality evaluation of a constructed repertoire in comparison with an ideal repertoire.
As an ideal repertoire you can use, for example, assembly of a sample, prepared using unique molecular identifiers(barcodes) to trace individual cDNA molecules. Using information about barcodes, near error-free assembly can be done with MiGEC tool and it can be used as an ideal repertoire. Also you can use simulated data from IgSimulator tool.

2. Installation

BarcodeQUAST supplied as a part of IgRepertoireConstructor tool and require same dependencies plus some additional Python libs. Full list of dependencies listed below:
  • 64-bit Linux system
  • g++ (version 4.7 or higher)
  • cmake (version 2.8.8 or higher)
  • Python (version 2.7 or higher), including:
    • BioPython
    • Matplotlib
    • NumPy
    • SciPy
  • zlib
  • libbz2
To install BarcodeQUAST, type: 
    
    ./prepare_cfg
	make

3. Usage

To run BarcodeQUAST, type: 

    ./barcode_quast.py [options] --Bc <barcode_clusters> -c <data_clusters> --out <output_dir>

3.1. Basic options:

-s <filename>
FASTQ file with Illumina reads (required).

-h/--help
show help message

--Bc <barcode_clusters>
Barcode sequences assembly in CLUSTERS.FA or MiGEC format (required).

--Br <barcode_clusters>
RCM(read-cluster mapping) file for barcodes assembly.

-c <data_clusters>
CLUSTERS.FA file for assembly to be evaluated (required).

-r <data_rcm>
RCM file for assembly to be evaluated (required).

--threads-num <threads_num>
Number of threads to use (by default 4).

--out<output_dir>
Output directory (required).

3.2. Advanced options:

--tau <tau>
Maximum allowed number of mismatches/gaps between sequences in barcodes and evaluated repertoire (by default 3).

--rate <rate>
Rate to distinguish between good and bad components (by default 0.5).

--deep-rcm-cmp
Align clusters from evaluated repertoire against corresponding barcode sequence and compute stats for such alignments. Computed only for good barcodes, recommended to with for only very high rate value (about 0.9) because of slow speed.

--rerun
Run with match results from the output directory.

3.3. Examples

To evaluate constructed repertoire with barcodes 
    
    ./barcode_quast.py --Bc barcodes.fastq -c constructed_clusters.fa -o output_dir

3.4. Output files

BarcodeQUAST creates working directory (which name was specified using option --out) and outputs there the following files:
  • metrics.txt - file with computed metrics.
  • cluster_sizes_distr.png - graph with distribution of cluster sizes (number of reads with the same barcode/in one constructed cluster).
  • cluster_sizes_distr_nt.png - graph with distribution of cluster sizes (number of reads with the same barcode/in one constructed cluster) for non-trivial clusters (size > 1).
  • isolated_cluster_sizes_distr.png - graph with distribution of cluster sizes (number of reads with the same barcode/in one constructed cluster) for isolated clusters (isolated cluster is a cluster that don't have a match in the second repertoire, a match is a cluster with sequence on distance <= tau).
  • isolated_cluster_sizes_distr_nt.png - graph with distribution cluster sizes for isolated non-trivial clusters.
  • lengths_distr.png - graph with cluster sequence lengths distribution
  • isolated_lengths_distr.png - graph with sequence lengths distribution for isolated clusters
  • barcode_quast.log - full log of barcodeQUAST run.

2. Metrics

Metrics computed for each provided repertoire:
  • #clusters - number of clusters (sequences) in repertoire.
  • #isolated clusters - number of clusters in repertoire that don't have similar cluster in the second repertoire (isolated cluster is a cluster that don't have a match in the second repertoire, a match is a cluster with sequence on distance <= tau).
  • #clusters (>= sz) - number of clusters of size >= sz (size of cluster is number of reads included to this cluster) in repertoire.
  • Min sequence length - minimal cluster sequence length.
  • Avg sequence length - average cluster sequence length.
  • Max sequence length - maximal cluster sequence length.
  • Min size of cluster - minimal cluster size.
  • Avg size of cluster - average cluster size.
  • Max size of cluster - maximal cluster size.
  • Min size of cluster - minimal size of isolated cluster.
  • Avg size of cluster - average size of isolated cluster.
  • Max size of cluster - maximal size of isolated cluster.
General metrics:
  • #good barcodes - number of barcodes having >= rate * barcode_size reads included to the same cluster in evaluated repertoire (barcodes covered by a single cluster in evaluated repertoire). This metric shows quality of reads clusterization in evaluated repertoire.
  • #bad barcodes - (#all barcodes - #good barcodes)
  • Min gap distance to good barcodes - minimal gap distance to good barcode computed among all clusters that share reads with this barcode. Cluster that covers > rate portion of reads in good barcode is excluded from the comparison. This metric is computed only if flag --deep-rcm-cmp is on.
  • Avg gap distance to good barcodes - average gap distance to good barcode computed among all clusters that share reads with this barcode. Cluster that covers > rate portion of reads in good barcode is excluded from the comparison. This metric is computed only if flag --deep-rcm-cmp is on.
  • Max gap distance to good barcodes - maximal gap distance to good barcode computed among all clusters that share reads with this barcode. Cluster that covers > rate portion of reads in good barcode is excluded from the comparison. This metric is computed only if flag --deep-rcm-cmp is on.
  • Min mismatches distance to good barcodes - minimal mismatches distance to good barcode computed among all clusters that share reads with this barcode. Cluster that covers > rate portion of reads in good barcode is excluded from the comparison. This metric is computed only if flag --deep-rcm-cmp is on.
  • Avg mismatches distance to good barcodes - average mismatches distance to good barcode computed among all clusters that share reads with this barcode. Cluster that covers > rate portion of reads in good barcode is excluded from the comparison. This metric is computed only if flag --deep-rcm-cmp is on.
  • Max mismatches distance to good barcodes - maximal mismatches distance to good barcode computed among all clusters that share reads with this barcode. Cluster that covers > rate portion of reads in good barcode is excluded from the comparison. This metric is computed only if flag --deep-rcm-cmp is on.
